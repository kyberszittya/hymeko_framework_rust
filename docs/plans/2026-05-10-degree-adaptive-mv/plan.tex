\section*{1. Motivation}

Three prior global-$K$ experiments (ABB, entropy heuristic, hybrid
$\alpha$-blend) all failed the HSiKAN smoke gate by $6{-}10$\,pp AUC
on Epinions.  The structural conclusion across those reports is:
\emph{per-vertex top-$m$'s value lies in its quantification — every
covered vertex contributes its locally-best $m$ cycles by signal —
which no global-$K$ scorer replicates regardless of how
signal $\times$ diversity is weighted}.

This plan accepts that finding and shifts the attack from
``replace per-vertex'' to ``speed up per-vertex'' while preserving
its selection structure.  Specifically: replace the fixed
$m_\text{per\_vertex} = 128$ cap with a per-vertex
\textbf{degree-adaptive} cap
\[
  m_v \;=\; \min\bigl(m_\text{max}, \;\max(m_\text{min}, \;c \cdot \deg(v))\bigr),
\]
parameterised by three knobs:
$m_\text{max}$ (current default $128$),
$m_\text{min}$ (lower bound; default $1$),
$c$ (slope, default $4$ — to be tuned).

\subsection*{1.1 Why this should help}

The per-vertex threshold probe at fixed $m{=}128$
(\texttt{examples/probe\_per\_vertex\_thresholds.rs}) showed:

\begin{itemize}
  \item $61\%$ of Epinions vertices are \emph{empty}: they touch zero
    $k{=}4$ balanced cycles. Wasted iteration in the rayon fold.
  \item $21\%$ are \emph{partial}: heap.len() $<$ 128, threshold $= 0$,
    ABB never fires at these vertices.
  \item $18\%$ are \emph{full}: threshold $\ge 0$, ABB potentially
    fires — but a cycle reaches a vertex's heap only if all $k$ of
    its vertices are full, and at uniform-vertex bound that's
    $0.1\%$ of cycles (probe-confirmed).
\end{itemize}

Setting $m_v$ in proportion to $\deg(v)$ has three coupled effects:

\begin{enumerate}
  \item \textbf{More vertices fill their heap.}  A vertex of degree
    $d$ touches roughly $\binom{d}{1}\cdot O(d^{k-1})$ candidate
    cycles in the worst case; in practice many fewer (BFS-pruned).
    With $m_v \propto d$, low-degree vertices have low caps that
    they can actually fill.  Predicted: full-heap rate climbs from
    $18\%$ to $\ge 60\%$ at $c = 4$ on Epinions
    (rough; verified empirically by the probe at the chosen $c$).
  \item \textbf{Per-vertex ABB becomes feasible.}  With more
    full-heap vertices, the joint probability that a candidate
    cycle's $k$ vertices are all full rises from $0.18^4 \approx
    0.001$ to $0.6^4 \approx 0.13$ at uniform-vertex bound; the
    actual rate is typically higher because cycles preferentially
    touch hubs (which fill faster).  ABB starts to prune real
    fractions of partial paths.
  \item \textbf{Total cycle output drops by a controlled factor.}
    High-degree hub vertices keep $m_\text{max} = 128$; low-degree
    leaves get small $m_v$.  The total cycle set is
    $\sum_v m_v \le \sum_v 128$ — strictly fewer.  The distribution
    is also \emph{degree-proportional}, which matches the empirical
    distribution of cycles in the graph (hubs participate in more
    cycles; leaves in fewer).
\end{enumerate}

\subsection*{1.2 Why this might fail}

\begin{itemize}
  \item \textbf{HSiKAN may rely on the fixed $m{=}128$ for high-degree
    vertices.}  If the model's per-row feature update assumes uniform
    $m$ across rows, a degree-proportional cap could shift the
    distribution badly.  The smoke test answers this.
  \item \textbf{$c$ is a new hyperparameter.}  Tuning over $c \in
    \{1, 2, 4, 8, 16\}$ adds 5 single-seed runs.  Acceptable.
  \item \textbf{The cycle distribution change is paper-relevant.}
    Any production promotion needs 5-seed paired AUC on the full
    config (c2,c3,c4,c5,w2,w3, h=4, 80 epochs), per the
    ``\texttt{n-seed-before-paper-headline}'' memory rule.
\end{itemize}

\section*{2. Scope and goal}

\textbf{In scope:}

\begin{itemize}
  \item \texttt{enumerate\_top\_k\_per\_vertex\_cycles\_par} accepts
    a per-vertex $m_v$ vector instead of a scalar
    $m_\text{per\_vertex}$.  Existing scalar-$m$ entry stays as a
    thin wrapper that fills $m_v[\cdot] = m_\text{per\_vertex}$.
    No call site breaks.
  \item New constructor helper
    \texttt{degree\_adaptive\_m\_v(graph, m\_min, m\_max, c) -> Vec<u32>}
    in \texttt{hymeko\_graph::topk\_cycles}.
  \item Per-vertex ABB stub: when at least one vertex on the
    candidate cycle has heap full, compute the threshold
    $\min$ across full-heap cycle vertices.  \emph{Skip} the
    full-vertex-min check if any vertex is non-full (cycle is
    auto-retained).  Loose admissibility but profile-justified: with
    most vertices full, the minority of non-full ones are usually
    leaves where any cycle is welcome.
  \item PyO3 binding extension: add \texttt{m\_min: usize},
    \texttt{m\_max: usize}, \texttt{m\_slope\_c: f64} args to
    \texttt{enumerate\_top\_k\_per\_vertex\_cycles\_signed\_rs}
    (default $m_\text{min} = m_\text{max} = m_\text{per\_vertex}$,
    $c = 0$, which collapses to the existing fixed-$m$ behaviour).
  \item Python env-vars:
    \texttt{HSIKAN\_TOPK\_M\_V\_C} (slope, defaults to 0 = fixed),
    \texttt{HSIKAN\_TOPK\_M\_V\_MIN} (defaults to 1),
    \texttt{HSIKAN\_TOPK\_M\_V\_MAX} (defaults to current
    \texttt{HSIKAN\_TOPK\_K} value).
\end{itemize}

\textbf{Out of scope:}

\begin{itemize}
  \item Reachability-based ABB (per-cycle prune that uses BFS-distance
    + remaining-vertex-set heuristics).  Future work.
  \item Combining with the hybrid $\alpha$-blended scorer.  Per-vertex
    structure preservation is the goal here; $\alpha$-blend changes
    the per-vertex scoring criterion, which we want fixed.
  \item Hypergraph-degree weighting (e.g. weighting vertex degree by
    its triadic count instead of edge count).  Worth an extension if
    raw degree proves a poor proxy.
\end{itemize}

\subsection*{2.1 Goal}

\textbf{Smoke gate (single seed, abbreviated config):}
\begin{itemize}
  \item \textbf{AUC within $\pm 0.02$ of fixed-$m{=}128$ baseline}
    (currently $0.6416$ for c3+c4 on Epinions, h=4, 20 epochs) at
    \emph{some} $c \in \{1, 2, 4, 8, 16\}$.  This is the binary
    go/no-go.
  \item Wall time $\le 60$\,s (vs $148.6$\,s baseline) at the
    same $c$.  Speed-up of at least $2.5\times$ end-to-end.
  \item Probe: at the chosen $c$, full-heap-vertex rate $\ge 50\%$
    on Epinions (vs current $18\%$).
\end{itemize}

\textbf{Promotion gate (5-seed paired, full production config):}

If smoke passes:

\begin{itemize}
  \item Production config: c2,c3,c4,c5,w2,w3, $h{=}4$, 80 epochs,
    balance pruner, per-vertex top-K with degree-adaptive $m_v$ at
    the chosen $c$.
  \item 5 seeds × \{fixed $m{=}128$ baseline, degree-adaptive at
    chosen $c$\} = 10 runs.
  \item Acceptance: paired-mean AUC within $\pm 0.005$ of fixed-$m$
    baseline; paired-$\sigma$ within $\pm 1.0\sigma$.  Wall-time win
    $\ge 2\times$ on the production config.
  \item Below threshold $\Rightarrow$ ship as research tool, document
    the gap.
\end{itemize}

\section*{5. Interface changes}

\begin{lstlisting}
/// Build a per-vertex m_v vector from the graph's degree
/// distribution.
///
///     m_v[v] = min(m_max, max(m_min, c * deg(v)))
///
/// `c = 0` recovers fixed m_v = m_min for every vertex; useful for
/// testing the adaptive path with a uniform vector.
///
/// `c >= 1` introduces degree-proportional caps; low-degree
/// vertices get small caps they can actually fill.
pub fn degree_adaptive_m_v(
    graph: &SignedGraph,
    m_min: u32,
    m_max: u32,
    c: f64,
) -> Vec<u32>;

/// Per-vertex top-K with a per-vertex cap vector.  `m_v.len()`
/// must equal `graph.n_nodes`.
///
/// At each vertex `v`, retain the top-`m_v[v]` cycles passing
/// through `v` by `score`.  Output is the deduplicated union.
pub fn enumerate_top_k_per_vertex_cycles_par_adaptive<P, S>(
    graph: &SignedGraph,
    k_len: usize,
    pruner: &P,
    m_v: &[u32],
    score: S,
) -> Vec<TopKCycle>
where
    P: CyclePruner + Sync,
    S: Fn(&[u32], &[i8]) -> f64 + Sync;
\end{lstlisting}

The fixed-$m$ entry point becomes:

\begin{lstlisting}
pub fn enumerate_top_k_per_vertex_cycles_par<P, S>(
    graph: &SignedGraph,
    k_len: usize,
    pruner: &P,
    m_per_vertex: usize,
    score: S,
) -> Vec<TopKCycle>
where ... {
    let m_v = vec![m_per_vertex as u32; graph.n_nodes as usize];
    enumerate_top_k_per_vertex_cycles_par_adaptive(
        graph, k_len, pruner, &m_v, score,
    )
}
\end{lstlisting}

\section*{6. Test strategy}

\subsection*{Unit (\texttt{topk\_cycles.rs::tests})}

\begin{itemize}
  \item \texttt{degree\_adaptive\_m\_v} formula correctness on a
    hand-checked fixture.
  \item Boundary: \texttt{c = 0} produces uniform $m_\text{min}$
    vector; \texttt{c} large produces clamp at $m_\text{max}$.
\end{itemize}

\subsection*{Integration (\texttt{tests/per\_vertex\_adaptive.rs})}

\begin{itemize}
  \item \textbf{Uniform-vector parity}: with $m_v[\cdot] = 8$ for
    every $v$, output equals
    \texttt{enumerate\_top\_k\_per\_vertex\_cycles\_par(\_, k=4,
    \_, 8, \_)}.  Same canonical cycle multiset (modulo tie-break).
  \item \textbf{Adaptive cap respect}: on a small fixture with
    known cycle structure, the output cycles touching a vertex $v$
    is at most $m_v[v]$.
  \item \textbf{Full-heap rate $\ge 50\%$ at $c \ge 2$}: on a
    150-vertex Erdős--Rényi fixture, the post-enumeration full-heap
    rate jumps from the fixed-$m$ rate to $\ge 50\%$ at $c=2$.
    Probe-confirmed via the threshold-reconstruction helper.
\end{itemize}

\subsection*{Performance (informational)}

Criterion bench at $c \in \{0, 1, 2, 4, 8, 16\}$ on a
5\,000-vertex synthetic graph: 5-iter median + IQR + worst per
CLAUDE.md \S3.  Track wall time, full-heap rate, total cycle count.
No asserted budget; informational characterisation.

\subsection*{Smoke gate (single-seed Epinions, abbreviated)}

For each $c \in \{1, 2, 4, 8, 16\}$:

\begin{itemize}
  \item Run with $m_\text{min} = 1$, $m_\text{max} = 128$,
    \texttt{HSIKAN\_TOPK\_M\_V\_C}=$c$, abbreviated config (c3+c4,
    h=4, 20 epochs, balance pruner).
  \item Record: total wall, AUC, F1, full-heap rate at the $c$,
    total cycle count.
\end{itemize}

\textbf{Pass}: any $c$ achieves AUC $\ge 0.6216$ (within $\pm 0.02$
of fixed-$m$ baseline 0.6416) AND wall $\le 60$\,s.

\textbf{Fail behaviour}: ship as research tool, document the curve;
two fallbacks recorded for the next iteration:

\begin{itemize}
  \item Try $m_\text{min} = 32$ instead of $1$ (give every vertex
    a non-trivial floor).
  \item Try a non-linear function: $m_v = \min(m_\text{max}, c
    \cdot \log\deg(v))$ — log-degree gives more cap to mid-degree
    vertices, less to extreme hubs.
\end{itemize}

\subsection*{Promotion gate (only if smoke passes)}

5-seed paired AUC at full production config:

\begin{itemize}
  \item Seeds: 0, 1, 2, 3, 4.
  \item Variants: \{fixed $m=128$ baseline, adaptive at chosen $c$\}.
  \item Production config: \texttt{HSIKAN\_MIXED\_TUPLES=c2,c3,c4,c5,w2,w3},
    $h=4$, 80 epochs, balance pruner, edge\_cr highway gate (the
    SOTA 5-seed config from
    \texttt{project\_epinions\_edge\_cr\_null\_2026\_05\_10}).
  \item Acceptance:
    \begin{itemize}
      \item Paired-mean AUC within $\pm 0.005$ of baseline.
      \item Paired $\sigma$ within $\pm 1.0\sigma$.
      \item Wall-time win $\ge 2\times$ on the production config.
    \end{itemize}
  \item Below threshold $\Rightarrow$ ship as research tool, no
    paper-headline claim.
\end{itemize}

\section*{8. Rollback path}

Strictly additive: \texttt{degree\_adaptive\_m\_v} +
\texttt{enumerate\_top\_k\_per\_vertex\_cycles\_par\_adaptive} are
new symbols.  The existing
\texttt{enumerate\_top\_k\_per\_vertex\_cycles\_par} becomes a thin
wrapper that delegates with a uniform $m_v$ vector — behavioral
parity is asserted in the unit test.  PyO3 default args
(\texttt{c} = 0) preserve the existing
\texttt{enumerate\_top\_k\_per\_vertex\_cycles\_signed\_rs}
behaviour.

\textbf{Single revert restores prior state}: no call site changes.

\textbf{Halt conditions} (CLAUDE.md \S10):

\begin{itemize}
  \item Uniform-vector-parity test fails $\Rightarrow$ revert; the
    refactor is broken.
  \item Smoke gate fails at every $c$ in the 5-point sweep
    $\Rightarrow$ ship as research tool; document the curve;
    queue the two fallbacks ($m_\text{min}=32$ floor; log-degree
    formula) as a new plan.
  \item 5-seed paired gate fails $\Rightarrow$ ship as research
    tool; document smoke-vs-full-config gap.
  \item Full-heap rate doesn't climb above 50\% at any $c$
    $\Rightarrow$ the threshold-probe assumption is wrong; halt and
    re-evaluate the per-vertex ABB integration plan.
\end{itemize}

\section*{9. Reporting requirements}

The smoke-gate and promotion-gate sections of the report must
include matplotlib plots (PNG, committed to
\texttt{reports/figures/}) for any gate-function metric swept over
a parameter:

\begin{itemize}
  \item AUC vs $c$ across the smoke sweep, with the gate band
    ($\pm 0.02$ around the fixed-$m$ baseline) shaded; the gate's
    pass/fail line marked.
  \item Wall time vs $c$.
  \item Full-heap rate vs $c$.
  \item Total cycle count vs $c$.
\end{itemize}

ASCII-art curves embedded in the markdown report are insufficient.
A single multi-panel figure summarising the gate functions side by
side is the right form factor; matplotlib's
\texttt{tight\_layout()} + \texttt{savefig(dpi=130)} produces a
report-grade output without further tuning.  Generation script
lives at \texttt{reports/figures/<slug>\_plot.py}; its output
\texttt{<slug>\_gates.png} is referenced from the markdown via
\texttt{![](figures/<slug>\_gates.png)}.
