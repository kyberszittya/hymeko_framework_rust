\section*{Motivation}

Two prior plans exposed two endpoint heuristics for HSiKAN's $M_e$
cycle source on Epinions:

\begin{itemize}
  \item \textbf{Global ABB} (\texttt{enumerate\_top\_k\_cycles\_par\_bb}
    + \texttt{FractionNegativeScorer}): selects the highest-$\,\mathrm{fraction\_negative}$
    cycles. Wall: $\sim 5$\,s. AUC: $0.5755$ (single seed). All
    cycles are score-1.0 all-negative balanced; $M_e$ has no score
    variance and biased vertex coverage.
  \item \textbf{Entropy heuristic}
    (\texttt{enumerate\_top\_k\_cycles\_par\_entropy} +
    \texttt{EntropyGainScorer}): selects cycles to maximise per-vertex
    incidence-distribution entropy. Wall: $\sim 17$\,s. AUC: $0.5396$
    (single seed). Vertex-uniform but signal-poor: $F_1$ collapses
    to 0.27 because retained cycles don't carry class-discriminating
    sign structure.
\end{itemize}

The per-vertex baseline at AUC $0.6416$ achieves both: vertex-uniform
\emph{and} high-signal-per-vertex, at the cost of $\sim 110$\,s
enumeration wall on Epinions $k{=}4$.

This plan proposes the natural hybrid: a single scorer
\[
  \mathrm{score}_\alpha(c \mid C) \;=\;
  \alpha \cdot \mathrm{signal}(c) \;+\; (1 - \alpha) \cdot
  \Delta H(c \mid C),
\]
parameterised at runtime by $\alpha \in [0, 1]$, with $\alpha{=}0$
recovering pure entropy and $\alpha{=}1$ recovering pure ABB.  The
intermediate sweep maps the \emph{signal $\times$ diversity Pareto
frontier} on real workloads — the empirical question is whether
\emph{any} interior $\alpha$ recovers per-vertex-baseline AUC at
ABB-class wall time.

\section*{Scope and goal}

\textbf{In scope:}
\begin{itemize}
  \item New trait-blanket impl \texttt{HybridScorer<B, H>}
    parameterised by:
    \begin{itemize}
      \item \texttt{B: BoundedScorer} — the signal component (existing
        \texttt{FractionNegativeScorer} / \texttt{BalanceScorer} /
        \texttt{SignProductAbsScorer} / \texttt{LowRootScorer}).
      \item \texttt{H: UniformityHeuristic} — the diversity component
        (existing \texttt{EntropyGainScorer} or
        \texttt{InverseDegreeScorer}).
      \item $\alpha \in [0, 1]$ — runtime mixing weight.
    \end{itemize}
    Implements \texttt{UniformityHeuristic} (the richer trait, since
    the diversity term is state-dependent); state-update / rollback
    delegate to the \texttt{H} component.
  \item Admissible UB by \emph{linearity of upper bounds}:
    \[
      \mathrm{UB}_\alpha = \alpha \cdot \mathrm{UB}_\mathrm{signal}
      + (1 - \alpha) \cdot \mathrm{UB}_\mathrm{div}.
    \]
    If both component UBs are admissible (the prior plans assert
    this), the linear combo is admissible by Jensen's inequality.
  \item No new entry point — reuse
    \texttt{enumerate\_top\_k\_cycles\_par\_entropy} (already takes a
    \texttt{UniformityHeuristic}); \texttt{HybridScorer} \emph{is} a
    \texttt{UniformityHeuristic}.
  \item PyO3 binding extension: add \texttt{alpha: f64} parameter to
    \texttt{enumerate\_top\_k\_cycles\_signed\_entropy\_rs}
    (default $\alpha = 0$ preserves existing behaviour) and a
    \texttt{signal\_kind} parameter selecting the \texttt{B} type.
  \item Python route: \texttt{HSIKAN\_TOPK\_HYBRID\_ALPHA} env-var
    (defaults to 0 = pure diversity); \texttt{HSIKAN\_TOPK\_SIGNAL}
    selects the signal scorer.
\end{itemize}

\textbf{Out of scope:}
\begin{itemize}
  \item Per-vertex hybrid (the per-vertex pipeline already mixes
    signal + uniformity by construction; this plan is for the
    global-K variants).
  \item Search over $\alpha$ inside the enumerator (we sweep $\alpha$
    \emph{outside} the call as a hyperparameter; no inner-loop
    optimisation).
  \item Degree-adaptive $m_v$ — that's a separate plan (the original
    plan B framing).
\end{itemize}

\section*{Interface changes}

\begin{lstlisting}
/// Linear blend of a signal scorer and a diversity heuristic.
///
/// `score(c, state) = alpha * signal(c) + (1 - alpha) * H_div(c, state)`
///
/// alpha in [0, 1]; alpha = 1 collapses to pure-signal global ABB,
/// alpha = 0 collapses to the pure-diversity entropy / inverse-degree
/// path.  Intermediate alpha maps the score x diversity Pareto frontier.
///
/// # Admissibility
///
/// UB_hybrid(prefix, k_rem, state) = alpha * UB_signal(prefix, k_rem)
///   + (1 - alpha) * UB_div(prefix, k_rem, state).
///
/// Linear combinations of admissible UBs are admissible: for any
/// realisable completion c of the partial state,
///   score_hybrid(c) = alpha * signal(c) + (1 - alpha) * H_div(c)
///                  <= alpha * UB_signal      + (1 - alpha) * UB_div
///                  =  UB_hybrid.
pub struct HybridScorer<B, H>
where
    B: BoundedScorer,
    H: UniformityHeuristic,
{
    pub signal: B,
    pub diversity: H,
    pub alpha: f64,
}

impl<B, H> HybridScorer<B, H>
where
    B: BoundedScorer,
    H: UniformityHeuristic,
{
    pub fn new(signal: B, diversity: H, alpha: f64) -> Self {
        debug_assert!((0.0..=1.0).contains(&alpha));
        Self { signal, diversity, alpha }
    }
}

impl<B, H> UniformityHeuristic for HybridScorer<B, H>
where
    B: BoundedScorer,
    H: UniformityHeuristic,
{
    fn score(&self, vs: &[u32], signs: &[i8],
             state: &UniformityState) -> f64 {
        let n_neg = signs.iter().filter(|&&s| s < 0).count();
        let s_signal = self.signal.score(vs, signs);
        let s_div = self.diversity.score(vs, signs, state);
        self.alpha * s_signal + (1.0 - self.alpha) * s_div
    }

    fn update(&self, vs: &[u32], signs: &[i8],
              state: &mut UniformityState) {
        // Only the diversity component owns state; signal is stateless.
        self.diversity.update(vs, signs, state);
    }

    fn rollback(&self, vs: &[u32], signs: &[i8],
                state: &mut UniformityState) {
        self.diversity.rollback(vs, signs, state);
    }

    fn upper_bound(&self, prefix_vs: &[u32], k_remaining: usize,
                   k_len: usize, state: &UniformityState) -> f64 {
        // Component bounds.  The signal UB is sign-conditional (it
        // takes n_neg_so_far as an arg).  At the partial-path
        // boundary the worst case for `signal` is "all remaining
        // edges negative" (n_neg_so_far = k_len - k_remaining for
        // fraction_negative), which is what BoundedScorer::upper_bound
        // delivers.  We pass n_neg = 0 here so the signal UB is
        // *fully optimistic* about future negative edges (admissible).
        let ub_signal = self.signal.upper_bound(0, k_len, k_len);
        let ub_div = self.diversity.upper_bound(
            prefix_vs, k_remaining, k_len, state,
        );
        self.alpha * ub_signal + (1.0 - self.alpha) * ub_div
    }
}
\end{lstlisting}

\textbf{Note on the signal UB.}  The current
\texttt{BoundedScorer::upper\_bound(n\_neg\_so\_far, k\_remaining, k\_len)}
takes \emph{partial-path} sign info, but \texttt{HybridScorer}'s
\texttt{UniformityHeuristic::upper\_bound} doesn't have access to
the signs of the prefix's edges (the trait passes \texttt{prefix\_vs}
only).  Approach: pass $n_{\mathrm{neg}} = 0$ at call time —
\emph{loose but admissible} — so the signal UB is the loosest
possible (best case = all remaining edges negative).  The DFS body
can be extended in a follow-up pass to thread $n_{\mathrm{neg}}$
through \texttt{dfs\_entropy}; deferred so this plan stays minimal.

\section*{Test strategy}

\subsection*{Unit (in \texttt{topk\_cycles.rs::tests})}

\begin{itemize}
  \item \textbf{Boundary collapse}: at $\alpha = 1$, \texttt{HybridScorer}'s
    \texttt{score}, \texttt{update}, \texttt{rollback},
    \texttt{upper\_bound} all collapse to the signal scorer's
    contribution alone (diversity term has weight 0).  At $\alpha = 0$,
    they collapse to the diversity scorer alone.  A 6-vertex hand
    fixture exercises both endpoints.
  \item \textbf{Linearity of UB}: numerical check on 100 random
    states + cycles — measure
    $|\mathrm{UB}_\alpha - (\alpha \cdot \mathrm{UB}_1 + (1 - \alpha)
    \cdot \mathrm{UB}_0)| < 10^{-9}$.
\end{itemize}

\subsection*{Integration (\texttt{tests/hybrid\_topk.rs})}

\begin{itemize}
  \item \textbf{$\alpha = 0$ parity vs entropy}: on a 200-vertex
    Erdős--Rényi fixture,
    \texttt{enumerate\_top\_k\_cycles\_par\_entropy} called with
    \texttt{HybridScorer\{ alpha: 0, ...\}} produces the same canonical
    cycle set (within tie-tolerance) as called with
    \texttt{EntropyGainScorer} directly.
  \item \textbf{$\alpha = 1$ parity vs ABB}: on the same fixture,
    \texttt{HybridScorer\{ alpha: 1, ...\}} produces the same canonical
    cycle SET as
    \texttt{enumerate\_top\_k\_cycles\_par\_bb(\&FractionNegativeScorer)}.
    Tie-breaking may differ (the entropy enumerator's heap uses
    different update semantics), so test asserts $\ge 90\%$
    canonical-cycle agreement.
  \item \textbf{Output cardinality}: at $\alpha \in \{0, 0.25, 0.5,
    0.75, 1\}$ on a 150-vertex fixture, output size $\le K$ exactly.
  \item \textbf{Admissibility on random fixtures}: 100 random
    partial-path states $\times$ 5 $\alpha$ values, every reachable
    completion's score $\le$ UB.
\end{itemize}

\subsection*{Performance (informational only)}

No asserted numerical budget.  Criterion bench on a 5\,000-vertex
synthetic graph at the 5 $\alpha$ values; report median + IQR + worst.
Goal is to characterise wall-time-vs-$\alpha$ trade-off, not enforce
it.

\subsection*{Smoke gate (single-seed Epinions)}

Run the abbreviated config (c3+c4, h=4, 20 epochs, K=10\,000,
\texttt{balance} pruner, \texttt{signal\_kind=fraction\_negative})
at $\alpha \in \{0.0, 0.25, 0.5, 0.75, 1.0\}$.  For each $\alpha$,
report total wall + AUC + macro-$F_1$.

\textbf{Promotion criterion}: if any $\alpha$ achieves AUC within
$\pm 0.02$ of the per-vertex baseline ($0.6416$ on Epinions
abbreviated config) — i.e. AUC $\ge 0.6216$ — promote that $\alpha$
to a 5-seed paired validation per the
\texttt{n-seed-before-paper-headline} memory rule.

\textbf{Fallback}: if no $\alpha$ in the 5-point sweep clears the
band, the plan halts at the smoke gate.  Two fallback options
documented in the report:
\begin{itemize}
  \item Finer $\alpha$ grid (say 11-point) — possible but unlikely to
    matter if the 5-point trend is monotonic and below the band.
  \item Pivot to degree-adaptive $m_v$ (the original plan B framing) —
    the per-vertex path's structure is what HSiKAN actually wants;
    speed it up rather than try to mimic it from global-K side.
\end{itemize}

\subsection*{5-seed paired validation gate (only if smoke passes)}

Production config (c2,c3,c4,c5,w2,w3, h=4, 80 epochs, balance
pruner) on Epinions, 5 seeds.  Acceptance:
\begin{itemize}
  \item Paired-mean AUC for the chosen $\alpha$ within $\pm 0.005$ of
    per-vertex $m{=}128$ baseline ($0.846 \pm 0.011$ from the prior
    \texttt{project\_epinions\_edge\_cr\_null} memory).
  \item Paired-$\sigma$ within $\pm 1.0\sigma$ of the baseline.
  \item Wall-time win documented (target $\ge 5\times$ over per-vertex
    enumeration on the production config).
\end{itemize}

\section*{Rollback path}

Strictly additive: \texttt{HybridScorer} is a new generic struct;
the four existing \texttt{BoundedScorer} impls and the two
\texttt{UniformityHeuristic} impls are unchanged.  \texttt{HybridScorer}
\emph{is} a \texttt{UniformityHeuristic}, so it plugs into the
existing \texttt{enumerate\_top\_k\_cycles\_par\_entropy} entry point
without a new enumerator.

\textbf{Rollback}: single-revert restores prior state.  No call sites
break.  PyO3 default values preserve the existing
\texttt{enumerate\_top\_k\_cycles\_signed\_entropy\_rs} behaviour at
$\alpha = 0$ + \texttt{signal\_kind}-irrelevant.

\textbf{Halt conditions} (CLAUDE.md §10):
\begin{itemize}
  \item Admissibility test fails on any $\alpha$ → revert, fix the UB
    derivation, re-test before re-trying.
  \item Smoke gate fails at every $\alpha$ → ship as research tool;
    pivot to degree-adaptive $m_v$ plan.
  \item 5-seed paired gate fails → ship as research tool; document
    smoke vs full-config gap.
\end{itemize}
