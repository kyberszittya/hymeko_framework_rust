The gap to SGT is $0.094$ at the 5-seed level.  If the adaptive $m_v$
delta scales fully ($+0.067$), production-config AUC would land at
$\approx 0.913$ — still $0.028$ below SGT.  We need additional

and the experimental order to test them, gated by the in-flight
5-seed result.  Each lever gets its own implementation plan only
once it passes a single-seed smoke test on the production config.

\subsection*{2.2 Detailed lever rationale}

\paragraph{A. Sub-unit $c$.}  The smoke c-sweep was monotonic
decreasing in AUC as $c$ grew from 1 to 16.  Extrapolating the
trend, $c < 1$ should lift further.  At $c{=}0.5$, low-degree
vertices (deg $\le 1$) get $m_v = 1$; mid-degree ($\deg{=}13$ avg
on Epinions) get $m_v = 7$; hubs still saturate at 128.  This
sharpens the signal-density argument from the smoke report further.
\textbf{Expected lift floor}: $+0.02$ over $c{=}1$ (i.e.\ on top of
the in-flight 5-seed result).  \textbf{Cost}: 3 smoke runs at
$c \in \{0.25, 0.5, 0.75\}$ at the abbreviated config.

\paragraph{B. Adaptive $m_v$ + sparse attention.}  Per
\texttt{project\_sparse\_attention\_synth\_2026\_05\_10}, the sparse
attention (\texttt{HSIKAN\_SPARSE\_ATTN\_K=8}) is synthetic-validated
but untested on real Epinions.  The per-edge attention sparsifier is
orthogonal to the cycle source change; combining both should be
near-additive.  \textbf{Expected lift}: $+0.01$ to $+0.03$.
\textbf{Cost}: 1 smoke run.

\paragraph{C. Davis pruner instead of balance.}  Currently
\texttt{HSIKAN\_TOPK\_PRUNER=balance} (Cartwright--Harary balanced
cycles only).  Davis weak balance keeps all cycles \emph{except}
all-negative triangles — a larger cycle pool with different sign
structure.  Smoke would tell whether the larger pool informs
HSiKAN better than the strict balanced subset.  \textbf{Risk}: low
(orthogonal to $m_v$).  \textbf{Cost}: 1 smoke.

\paragraph{D. $m_\text{max}$ sweep.}  We've held $m_\text{max} = 128$
throughout.  Combined with adaptive $c{=}0.5$, low-degree vertices
get tiny caps already, so reducing $m_\text{max}$ from 128 to 64 or
32 only affects hubs.  If hubs are the over-saturation source, this
should help further.  \textbf{Cost}: 3 smoke runs at $m_\text{max}
\in \{32, 64, 128\}$.

\paragraph{E--H.}  Lower-priority; tested only after A--D land.

The natural \textbf{stacking order} is A $\to$ B $\to$ C, choosing
the best of each before stacking the next.  Each stack-step requires
its own 5-seed paired validation.

\section*{3. Sweep plan}

\subsection*{3.1 Pre-5seed CPU-only studies (this plan)}

While the in-flight 5-seed runs (~3.8\,h), launch CPU-only
cycle-distribution probes that don't compete for the GPU:

\begin{itemize}
  \item Extend \texttt{probe\_per\_vertex\_thresholds.rs} with a
    \texttt{c} CLI argument; sweep $c \in
    \{0.25, 0.5, 1.0, 2.0, 4.0, 8.0\}$.
  \item Report per $c$: full-heap-rate, total cycle count, vertex
    coverage histogram, score distribution.
  \item Generate a matplotlib plot to visualize the trend.
  \item Predict the cycle-source landscape we'd train on at each $c$.
\end{itemize}

This gives us a data-grounded prediction for which $c$ value to test
next at training-time, ahead of running the smoke.

\subsection*{3.2 Post-5seed sweep order}

Gated on the in-flight 5-seed result:

\begin{enumerate}
  \item \textbf{If} adaptive $c{=}1$ at production passes the
    $\pm 0.005$ AUC band over fixed-$m$ baseline: confirmed; proceed
    to A--D.
  \item Run lever A (sub-unit $c$) single-seed smokes at
    $c \in \{0.25, 0.5, 0.75\}$.  Pick the best.  Run 5-seed at the
    best $c$ for promotion.
  \item Run lever B (sparse attn) single-seed at the best $c$.  Pick
    the best $K_\text{attn}$ value.  5-seed at best.
  \item Run lever C (Davis pruner) single-seed at the best $c$,
    best $K_\text{attn}$.  5-seed.
  \item Run lever D ($m_\text{max}$ sweep) single-seed at the chosen
    A+B+C combination.  5-seed.
\end{enumerate}

Each step gates the next: if the smoke fails, halt that branch and
move to the next lever in the matrix.

\section*{4. Smoke and promotion gates}

\subsection*{Smoke gate (single-seed, abbreviated config)}

For each candidate lever:

\begin{itemize}
  \item AUC $\ge$ current best - 0.01 (allow for single-seed noise).
  \item Wall time $\le 2 \times$ baseline at the matching config.
  \item No regression in macro-$F_1$ below 0.50.
\end{itemize}

Pass $\Rightarrow$ promote to 5-seed paired.

\subsection*{Promotion gate (5-seed paired, production config)}

Same protocol as the in-flight degree-adaptive 5-seed:

\begin{itemize}
  \item Paired-mean AUC $\ge$ current best with $\sigma \ge 1.0$.
  \item Paired $\sigma$ within $\pm 1.0\sigma$ of baseline.
  \item Per-seed delta consistent (no all-five-seed swings).
\end{itemize}

\section*{7. Halt conditions}

\begin{itemize}
  \item In-flight 5-seed shows adaptive $c{=}1$ regression $\ge 0.01$
    AUC at production config $\Rightarrow$ this whole sweep is
    quality-poisoned; revert and re-evaluate the smoke-result
    interpretation.
  \item Any single-seed smoke shows AUC drop $> 0.05$ vs current
    best $\Rightarrow$ halt that branch; do not 5-seed.
  \item GPU contention with in-flight 5-seed $\Rightarrow$ defer
    GPU smokes until the 5-seed completes.
\end{itemize}
