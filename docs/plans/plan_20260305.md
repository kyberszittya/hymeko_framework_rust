# Hymeko Engine: Master Architectural & Mathematical Blueprint

## I. Topological Extraction & Sparse Representations (Rust Core)
The foundational layer lowers the parsed Intermediate Representation (IR) into mathematically pristine, memory-aligned sparse tensors, avoiding the exponential memory bloat of standard graph projections.

### 1. Expansion Strategies
* **Bipartite Star Expansion (The Standard):** Preserves the exact hypergraph topology by modeling connections as a bipartite graph between nodes and hyperedges. Complexity scales linearly: $\text{NNZ} \approx V \times E \times \text{density}$. This is the native format for our HyperKAN.
* **Clique Expansion (The Fallback):** Projects hyperedges into fully connected subgraphs. It suffers from quadratic information explosion per hyperedge. Complexity scales as $O(d^2)$ where $d$ is the node degree. Used solely for legacy matrix-based spectral benchmarks.

### 2. Sparse Tensor Coalescing
The engine extracts non-zero elements (NNZ) and coalesces duplicate connections safely and efficiently.
* **CSR (Compressed Sparse Row - 2D):** Used for strict 2D adjacency. Coalescing is strictly row-bounded to leverage CPU cache lines. Instead of a global $O(\text{NNZ} \log \text{NNZ})$ sort, the engine sorts columns locally within each row boundary, yielding a complexity of $O(N \cdot d \log d)$ where $d$ is the average degree.
* **COO (Coordinate - 3D):** The absolute optimal format for multi-dimensional spatial mapping (e.g., $k$ for hyperedge, $i$ for source, $j$ for target). Coalesced via an unstable lexicographical sort ($O(\text{NNZ} \log \text{NNZ})$) packed into contiguous tuples `(usize, usize, usize, F)` to saturate L1 cache during alignment.

## II. Cyber-Physical Serialization Pipeline
A mathematically compressed serialization stack designed to bridge the physical world (via optics) with the in-memory Rust engine, achieving a ~66% payload reduction.

### The Conversion Flow
1.  **Payload Bundling:** The `Ir`, `Index` (path lookups), and the complete `Interner` string dictionary are serialized together to prevent dangling integer pointers (`SymId`).
2.  **Binary Translation:** Serialized to **CBOR** (Concise Binary Object Representation) to eliminate the verbose syntax of JSON.
3.  **Algorithmic Compression:** The CBOR byte array is pushed through **Zlib/Deflate** (Level 9) to obliterate high-entropy structural repetitions (e.g., repeating `dkinddNodedname` keys).
4.  **Optical Encoding:** The compressed binary is mapped to **URL-Safe Base64**, stripping out null bytes and non-printable characters to ensure absolute grid resilience for camera sensors.
5.  **Physical Target:** The resulting alphanumeric string fits comfortably within a **Version 17/18 QR Code**, scannable by standard mobile optics without severe focus hunting.

## III. Spectral Graph Theory & Entropy Evaluation
Analyzing the structural stability and information density of the hypergraph. We strictly avoid the $O(N^3)$ cost of exact eigendecomposition, opting for highly scalable stochastic and polynomial approximations.

### 1. Matrix Foundations
* **Normalized Hypergraph Laplacian:** $$\tilde{L} = I - D_v^{-1/2} A D_v^{-1/2}$$
* **Scaled Laplacian:** Chebyshev polynomials require a bounded spectrum $[-1, 1]$. We scale the Laplacian using the maximum eigenvalue (approximated via power iteration):
  $$L_{scaled} = \frac{2\tilde{L}}{\lambda_{max}} - I$$

### 2. Chebyshev Approximation of Von Neumann Entropy
We calculate the spectral entropy $S = -\text{Tr}(L \log L)$ by approximating the matrix function $f(L) = -L \log L$.
* **Stochastic Trace Estimation (Hutchinson's Trick):** We approximate the trace using random Rademacher vectors $z \in \{-1, 1\}$:
  $$\text{Tr}(f(L)) \approx \mathbb{E}[z^T f(L) z]$$
* **Chebyshev Recurrence Relation:** Executed natively via Sparse Matrix-Vector Multiplications (SpMV):
  * $T_0(L)z = z$
  * $T_1(L)z = L_{scaled} z$
  * $T_k(L)z = 2 L_{scaled} T_{k-1}(L)z - T_{k-2}(L)z$

### 3. Dirichlet Energy
Used as an "over-smoothing canary" to mathematically prove if the neural network is collapsing node features $X$ into an indistinguishable average.
$$E(X) = \text{Tr}(X^T L X)$$

## IV. Hypergraph Kolmogorov-Arnold Networks (HyperKAN)
Standard GNNs use fixed activation functions on the nodes. Our HyperKAN places learnable, highly expressive univariate functions on the *hyperedges*, leveraging the Bipartite Star topology.

### 1. NURBS Parameterization
We replace simple scalar weights with Non-Uniform Rational B-splines (NURBS) to perfectly represent complex, continuous conic geometries across the topological manifold.
$$f(x) = \frac{\sum w_i c_i B_i(x)}{\sum w_i B_i(x)}$$

### 2. The Positivity Guardrail
> **Critical Architectural Constraint:** Unconstrained gradient descent will push rational weights $w_i$ to zero, causing instantaneous division-by-zero panics (`NaN` gradients).

All learnable rational weights must pass through a strictly positive activation function before evaluating the denominator:
$$w_{safe} = \ln(1 + \exp(w_i)) + \epsilon$$

## V. Tensor-Native Fuzzy Logic & Query Operations
Graph queries are not evaluated through slow, pointer-chasing node traversals. They are executed directly in the GPU's linear algebra cores.

### 1. Topological Querying via SpMV
Using the incidence matrix $H$ and a one-hot query vector $q$:
* **Shared Hyperedges:** $e = H^T q$
* **Neighboring Nodes:** $n = H e$

### 2. Continuous Fuzzy Set Theory
Hyperedge membership is extended from binary $\{0, 1\}$ to a continuous domain $[0, 1]$, replacing standard message passing operators:
* **Gödel T-norm (Fuzzy AND / Intersection):** $T(x, w) = \min(x, w)$
* **Product T-norm:** $T(x, w) = x \cdot w$
* **Gödel S-norm (Fuzzy OR / Union):** Implemented via PyTorch's `scatter_reduce_` using the `amax` operator to aggregate the maximum fuzzy signal across shared nodes.
