# Development Roadmap: Hypergraph-KA Engine

## Phase 1: The Execution Kernel (GPU Mathematics)
Standard linear neural network layers cannot execute the multi-way Hyper-KA Operator.
* **Triton Kernel Implementation:** Develop a custom GPU kernel that natively reads the `row_ptr` and `col_ind` CSR arrays to evaluate the univariate B-splines directly on the sparse incidence connections.
* **SRAM Optimization:** Process the inner and outer Kolmogorov-Arnold sums directly within the GPU's SRAM to eliminate latency-heavy round-trips to global memory.

## Phase 2: The Neural Integration (PyTorch Bridge)
The raw execution kernel must be seamlessly packaged for the machine learning ecosystem.
* **Module Construction:** Wrap the Triton kernel inside a standard PyTorch `torch.nn.Module` (e.g., `HyperKANLayer`).
* **Autograd Binding:** Ensure the backward pass mathematically routes gradients through the sparse CSR topology to properly update the B-spline control points.

## Phase 3: The Empirical Benchmark (IEEE SMC Proof)
We must empirically prove the superiority of the dual-frequency architecture (Neural Masking + Epoch Recompilation) over standard Graph Neural Networks.
* **Dynamic Cyber-Physical Dataset:** Design a time-series simulation of a morphing system, such as a robotic arm experiencing mechanical failure or an IoT drone swarm dynamically reconfiguring its communication graph.
* **Latency Profiling:** Benchmark the recompilation latency of the Rust engine against Python-based alternatives (like PyTorch Geometric) to prove real-time (100Hz+) viability.

## Phase 4: The Database Substrate (Durability)
To scale the topological engine into a true Hypergraph Database, RAM-only execution is insufficient.
* **KV-Store Embedding:** Integrate a high-performance embedded key-value store (such as `redb` or `RocksDB`) directly beneath the Rust `Ir` arena.
* **ACID Transactions:** Implement write-ahead logging (WAL) to guarantee that high-frequency IoT telemetry does not corrupt the topological pointers during concurrent state writes.
