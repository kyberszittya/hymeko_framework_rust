\section*{Motivation}

The smoke test from the prior task
(\texttt{reports/2026-05-10-abb-global-topk.md}) found that
\emph{global} top-$K$ ABB delivers a $25\times$ wall-time reduction
on Epinions but \emph{does not} help HSiKAN training: at $K{=}10\,000$
all retained cycles are score-1.0 (all-negative balanced) — the
$M_e$ matrix has no score variance and no per-vertex coverage in the
long tail. Single-seed AUC dropped from $0.642$ (per-vertex $m{=}128$
baseline) to $0.575$ (global ABB at the matched abbreviated config).

The per-vertex top-$m$ enumerator works because it gives \emph{every}
vertex its top-$m$ cycles, producing a vertex-uniform $M_e$.  The cost
is the $O(n_{\text{vertices}} \cdot m)$ per-vertex heap traffic and
the BFS pre-pass that dominates $66\%$ of cycles in the post-CSR
profile.

This plan proposes a third path: select cycles to \emph{maximise
the entropy of the per-vertex incidence distribution}.  Vertices
that are already well-covered contribute less to the next cycle's
score; rare vertices contribute more.  The result is vertex-uniform
$M_e$ \emph{by construction}, paired with branch-and-bound pruning
to keep wall time low.

\section*{Definitions}

Let $C \subseteq \mathcal{C}_k$ be the current set of accepted
$k$-cycles.  Define the per-vertex incidence count
$c_v = |\{c \in C : v \in c\}|$ and the empirical incidence
distribution
\[
  p_v = \frac{c_v}{\sum_{w} c_w}.
\]
The Shannon entropy of the incidence distribution is
\[
  H(C) = -\sum_v p_v \log p_v,
\]
maximised at $p_v = 1 / |\{v : c_v > 0\}|$ (uniform over covered
vertices).

\textbf{Goal:} given a budget $K$, select $C^* \subseteq \mathcal{C}_k$
with $|C^*| = K$ that approximately maximises $H(C^*)$, possibly
subject to a pruner $P : \mathcal{C}_k \to \{\text{accept, reject}\}$.

\section*{Approach}

\subsection*{Greedy entropy-gain scoring}

For a candidate cycle $c \in \mathcal{C}_k$ relative to the current
$C$, the marginal entropy gain is
\[
  \Delta H(c \mid C) = H(C \cup \{c\}) - H(C).
\]
A standard greedy max-coverage argument gives a $(1 - 1/e)$
approximation to the optimum if $H$ is monotone submodular
(approximately true for vertex-coverage entropy until saturation).

\subsection*{State-dependent scorer}

Add a new trait \texttt{StatefulScorer} that takes the running
per-vertex counts $c$ and the candidate cycle:
\begin{lstlisting}
pub trait StatefulScorer: Sync + Send {
    /// Score this cycle given the current per-vertex incidence
    /// counts.  For entropy-heuristic: returns $\Delta$H(c | C).
    fn score(&self, vs: &[u32], signs: &[i8], counts: &[u32],
             total: u64) -> f64;
    /// Update the running counts to include this cycle.
    fn update(&self, vs: &[u32], counts: &mut [u32], total: &mut u64);
    /// Upper bound on score given partial state.  For entropy:
    /// use rare-vertex bound — score cannot exceed the entropy
    /// contribution of the LEAST-covered vertex among the cycle's
    /// committed prefix.
    fn upper_bound(&self, prefix_vs: &[u32], k_remaining: usize,
                   counts: &[u32], total: u64) -> f64;
}
\end{lstlisting}

\subsection*{Two concrete scorers}

\begin{enumerate}
  \item \texttt{EntropyGainScorer}: $\Delta H(c \mid C)$ as above.
    State-dependent, monotonically decreasing as $C$ grows.
  \item \texttt{InverseDegreeScorer} (state-independent baseline):
    \[
      \mathrm{score}(c) = \sum_{v \in c} \frac{1}{\sqrt{c_v + 1}},
    \]
    where the $\sqrt{}$ smooths the contribution.  No state update
    needed across DFS recursion (just read $c$); cheaper to bound.
\end{enumerate}

\subsection*{Branch-and-bound on entropy gain}

For \texttt{EntropyGainScorer}, the partial-path UB is:
\[
  \mathrm{UB}(\text{prefix}, k_{\text{rem}}, c, \text{total}) =
  \Delta H_{\text{prefix}} + k_{\text{rem}} \cdot \max_v
  \left[\Delta H_v\right]
\]
where $\Delta H_v$ is the entropy contribution of adding vertex $v$
to the running cycle, taking the maximum over all reachable future
vertices (loose; tighter via BFS-distance-restricted reachable set).

If \texttt{UB} $\le$ heap min, prune the subtree.  Threshold rises
as the heap fills with high-$\Delta H$ cycles → ABB fires
progressively.

\subsection*{Per-thread state for rayon}

A subtlety: \texttt{StatefulScorer} maintains running counts.  In
the rayon-parallel variant, each fold task gets its own count
vector; entropy is computed locally; reduce merges by re-scoring.
This trades exactness (a cycle that looks high-$\Delta H$ on thread
1 might be redundant after thread 2's set is merged in) for
parallelism.  Acceptable for a heuristic: the global greedy bound
is already approximate.

\section*{Scope and goal}

Add an entropy-heuristic top-$K$ enumerator alongside the existing
ABB / per-vertex paths.  Specifically:

\begin{itemize}
  \item New trait \texttt{StatefulScorer} in
    \texttt{hymeko\_graph::topk\_cycles}.
  \item Concrete impls: \texttt{EntropyGainScorer},
    \texttt{InverseDegreeScorer}.
  \item New entry points: \texttt{enumerate\_top\_k\_cycles\_entropy}
    (sequential) and \texttt{enumerate\_top\_k\_cycles\_par\_entropy}
    (rayon).
  \item Builder method: \texttt{TopKBuilder::entropy(k\_keep,
    scorer)}.
  \item PyO3 binding: \texttt{enumerate\_top\_k\_cycles\_signed\_entropy\_rs}.
  \item Python route: \texttt{HSIKAN\_TOPK\_MODE=entropy}.
\end{itemize}

\textbf{Goal:} on Epinions $k{=}4$, abbreviated training config
(c3+c4, $h{=}4$, 20 epochs):
\begin{itemize}
  \item AUC within $\pm 0.02$ of per-vertex $m{=}128$ baseline ($0.642$
    on this seed).  This is the \emph{information sufficiency} test.
  \item Wall time $\le 30$ s (vs $148.6$ s for per-vertex baseline).
    This is the \emph{speedup} test.
\end{itemize}
The 5-seed paired AUC validation per the
``\texttt{n-seed before paper-headline}'' memory rule is gated on
the smoke-test passing first.

\section*{Test strategy}

\subsection*{Unit (lib)}

\begin{itemize}
  \item \texttt{StatefulScorer::EntropyGainScorer} returns 0 on a
    fresh empty $C$ for cycles already seen, $> 0$ for cycles that
    add new vertices.
  \item \texttt{InverseDegreeScorer} is monotonic in counts.
  \item Entropy UB is admissible: for 1\,000 random partial paths,
    every reachable completion's actual $\Delta H$ is $\le$ UB.
\end{itemize}

\subsection*{Integration}

\begin{itemize}
  \item On a 200-vertex Erd\H{o}s--R\'enyi fixture, the entropy-
    heuristic output's per-vertex coverage variance is at most
    $1.5\times$ the per-vertex top-$m$ baseline's coverage variance
    (i.e. the entropy path achieves comparable uniformity).
  \item The output cardinality matches $K$ (or all reachable
    cycles, if fewer).
  \item Comparison with global ABB: entropy-heuristic produces a
    lower-mean-score, higher-coverage cycle set.
\end{itemize}

\subsection*{Performance}

\begin{itemize}
  \item Criterion bench on 5\,000-vertex synthetic graph,
    $k{=}4$, $K{=}10\,000$, balance pruner: \emph{informational}
    only — no asserted budget, since the goal is information
    quality, not raw speed.
  \item Real-data smoke: 1-seed Epinions training as in the prior
    task's smoke test, with \texttt{HSIKAN\_TOPK\_MODE=entropy
    HSIKAN\_TOPK\_K=10000}.  Report wall time + AUC vs the two
    baselines (per-vertex, global ABB).
\end{itemize}

\subsection*{HSiKAN promotion gate}

If the smoke test passes (AUC within $\pm 0.02$ of per-vertex
baseline AND wall $\le 30$ s), trigger 5-seed paired validation:
\begin{itemize}
  \item 5 seeds × \{per-vertex $m{=}128$, entropy K=10\,000,
    entropy K=100\,000\} on Epinions (the production-relevant
    dataset).  Production config: c2,c3,c4,c5,w2,w3, $h{=}4$,
    80 epochs, balance pruner.
  \item Acceptance: paired-mean AUC within $\pm 0.005$ of
    per-vertex baseline, paired-$\sigma$ within $\pm 1.0\sigma$.
    Below threshold $\Rightarrow$ entropy-heuristic ships as a
    research tool, not production.
\end{itemize}

\section*{Rollback path}

Strictly additive — \texttt{TopKBuilder::entropy} is a new method,
\texttt{StatefulScorer} is a new trait, the entropy-aware enumerators
are new entry points.  No existing behavior changes.  Single revert
restores prior state; the per-vertex baseline and global ABB tools
remain.

\textbf{Halt condition:} if the smoke-test AUC drops below the
$\pm 0.02$ band, the entropy-heuristic ships as a research artefact
only and is not promoted to HSiKAN production; the \texttt{plan B}
fallback (degree-adaptive $m_v$ in the existing per-vertex path) is
re-opened as a follow-up plan.
