\section*{Scope and goal}
\textbf{Scope:} signed link prediction with HymeKo-G\"omb
(\texttt{run\_gomb\_smoke}, \texttt{run\_gomb\_tune}) on the same datasets
and \texttt{80\_10\_10} split already used for external SGCN / HSiKAN
comparisons (Bitcoin Alpha/OTC, Slashdot, Epinions).

\textbf{Goal:} define a \emph{publishable} procedure to (i)~push mean test
AUROC/AP beyond agreed reference bars, and (ii)~report uncertainty and
significance so ``breaking'' a metric is not a single-seed cherry-pick.
\textbf{``Break''} means: mean test AUROC (or AP, if primary for a venue)
exceeds the reference by more than a pre-declared margin, with a
two-sided 95\% bootstrap CI (or DeLong-style AUC comparison) that excludes
equality.

\section*{Reference bars (locked targets)}
\begin{itemize}
  \item \textbf{Published SGCN} (Derr et al., SNAP Bitcoin graphs):
  AUROC $\sim$0.91--0.93 depending on graph (see
  \texttt{docs/plans/2026-05-12-gomb-external-auc-tuning/plan.tex}).
  \item \textbf{In-repo tuned SGCN} (draft/paper notes): e.g.\ Slashdot
  $\sim$0.914 under matched protocol; Bitcoin Alpha/OTC bars as in
  \texttt{PAPER\_DRAFT\_2026\_05\_03.md} / external-auc tuning report.
  \item \textbf{Current G\"omb joint} (single seed, 48 epochs, compact):
  Slashdot \texttt{test\_auroc}$\approx$0.896 (JSONL
  \texttt{reports/gomb\_tune\_slashdot\_joint\_auc\_2026\_05\_12.jsonl}).
  Gap to in-repo SGCN Slashdot $\approx$0.018 AUROC --- primary stretch target.
\end{itemize}

\section*{Protocol (anti-leakage)}
\begin{enumerate}
  \item \textbf{Hyperparameter search} uses only \textbf{validation} metrics
  (\texttt{val\_auroc}, \texttt{val\_average\_precision}) to rank trials during
  \texttt{run\_gomb\_tune}. The held-out \texttt{test\_*} fields are logged but
  \textbf{not} used for early stopping or search decisions in the strict mode.
  \item \textbf{Winner selection:} after search, fix \texttt{trial\_params} for
  the best \emph{val} row (not best test).
  \item \textbf{Generalisation phase:} rerun that configuration for
  \texttt{data\_seed} $\in \{0,\ldots,K-1\}$ with $K \geq 5$ (CLAUDE.md
  benchmark stability). Report \textbf{mean, std, median, IQR} of
  \texttt{test\_auroc} and \texttt{test\_average\_precision}.
  \item \textbf{Single-seed headline} numbers are diagnostic only; the
  headline claim uses the multi-seed aggregate.
\end{enumerate}

\section*{Staged compute plan}
\paragraph{Stage A --- Epoch / convergence curve (single seed).}
Hold \texttt{trial\_params} fixed (e.g.\ best val from a dense search).
Sweep \texttt{n\_epochs} $\in \{32,48,64,96,128\}$ (or until val plateaus).
Record \texttt{val\_auc\_best}, \texttt{test\_auroc} at final epoch.
\textbf{Budget:} $\leq 10\%$ of full multi-seed wall before queuing overnight.

\paragraph{Stage B --- Dense random / Sobol search (val objective).}
\texttt{run\_gomb\_tune} with $\geq 48$ trials per dataset envelope
(joint-mix SNAP compact; Bitcoin joint with existing clamps). Log
\texttt{n\_params}, inference throughput (already in smoke JSON).

\paragraph{Stage C --- Multi-seed evaluation.}
For each dataset, one frozen config from Stage~B; $K$ seeds; optional
\texttt{joint\_slot\_cap} ablation $\{8000,12000,16000,0\}$ on SNAP if VRAM
allows \texttt{0}.

\paragraph{Stage D --- Significance vs bar.}
Either: (i)~paired bootstrap on per-edge scores exported once (storage-heavy),
or (ii)~DeLong variance for AUROC if a reference model's per-edge scores exist;
if not, report bootstrap over $K$ seeds only (weaker claim: ``mean exceeds bar
by $\delta$ with CI not overlapping bar'').

\section*{Test strategy}
\begin{itemize}
  \item Unit: JSONL aggregation script on synthetic 5-line fixture (median,
  IQR deterministic).
  \item Integration: one CPU smoke with two \texttt{data\_seed} values and
  identical \texttt{trial\_params} hash in logs.
\end{itemize}

\section*{Performance budget}
Peak RSS $\leq 16$\,GB per subprocess (CLAUDE.md \S4). Slashdot joint remains
joint-slot subsampled by default; any increase in \texttt{topk} or removal of
cap requires a one-seed VRAM smoke before multi-seed.

\section*{Worst-case inputs}
Full Bitcoin joint tuple pools $\times$ wide \texttt{topk}; Slashdot without
\texttt{joint\_slot\_cap}; Epinions $|V|$ embedding + four slots. All capped by
existing smoke flags unless explicitly escalated.

\section*{Rollback}
Delete orchestration script and aggregate helper; keep JSONL artifacts read-only
for history.

\section*{Risk anticipation}
\textbf{Test leakage:} repeated peeking at \texttt{test\_auroc} during manual
search invalidates claims --- mitigated by val-only selection + frozen config.
\textbf{Subsampling bias:} \texttt{joint\_slot\_cap} changes the training
distribution; document cap in every table row. \textbf{Variance:} high
variance across \texttt{data\_seed} may prevent ``break'' despite a lucky
single seed --- mitigated by $K \geq 5$ and reporting IQR.
