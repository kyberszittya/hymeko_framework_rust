\paragraph{Motivation.}
The 2026-05-13 Bitcoin Alpha Optuna run converged on a sparse arity
support \texttt{\{c2, c5, w2, w3, w4\}} with \texttt{c3, c4} dropped.
Closure-bearing cycles at the \emph{endpoints} of the arity range and
walks covering the middle is a candidate ``complementarity'' pattern
worth probing at larger $n$.  Hard-coded \texttt{c2..c5} prevents this
test.  We need a general $c_n$ extension \emph{and} an optimization
pattern that does not collapse to ``higher $n$ always wins by
parameter count''.

\paragraph{Scope.}
Three-layer optimization for cycle-arity selection on the
\texttt{signedkan\_wip} stack.
\begin{description}
  \item[L1 -- Inner training.] Replace softmax/sigmoid $\alpha_k$ on
  the arity weight vector with a \textbf{Hard-Concrete L0} gate.
  Differentiable, pushed toward $\{0,1\}$, with a tunable expected-L0
  budget $\mathbb{E}[\|\alpha\|_0]$.
  \item[L2 -- Outer search.] Add an \textbf{ASHA / successive-halving
  pruner} to the Optuna driver, gated on partial-epoch AUC.
  Composes with \texttt{HSIKAN\_OPTUNA\_SKIP\_EXPENSIVE\_ATTENTION}.
  \item[L3 -- Enumeration budget.] Per-arity caps proportional to
  $C / H(p_n)$, where $H(p_n)$ is the empirical entropy of the
  per-vertex cycle-count distribution at arity $n$.  Stops $c_7$ from
  drowning $c_2$ by raw count.
\end{description}

\paragraph{Goal.}
A configurable arity range \texttt{HSIKAN\_ARITY\_RANGE="2,3,4,5,6,7"}
runnable on the existing Bitcoin / Slashdot / Epinions pipelines,
with L1-L3 gated behind env flags defaulting OFF (current behavior
preserved bit-for-bit).  Acceptance: after Bitcoin Alpha 5-seed paired
sweep, the learned Hard-Concrete gate identifies a sparse arity
support, and ASHA cuts wall time by $\geq 30\%$ vs the same 30-trial
TPE-only run at equivalent best AUC.

\paragraph{Hard-Concrete contract (L1).}
For arity weights $\boldsymbol{\alpha} \in \mathbb{R}^{|N|}$, sample
$g_k \sim \mathrm{HardConcrete}(\log\alpha_k, \beta, \gamma, \zeta)$
with stretch parameters $(\gamma, \zeta) = (-0.1, 1.1)$ and temperature
$\beta = 2/3$ (Louizos et al.\ 2018).  Loss adds
$\lambda_{L0} \sum_k \Pr[g_k > 0]$, a closed-form expression in
$\log\alpha_k$.  At inference, gates are deterministic
$\bar g_k = \min(1, \max(0, \sigma(\log\alpha_k)(\zeta-\gamma) + \gamma))$.

\paragraph{Equal-information caps (L3).}
For each arity $n$ in \texttt{HSIKAN\_ARITY\_RANGE}, run a cheap
sampled-count probe (5\% of vertices) to estimate per-vertex cycle
count $c_n(v)$.  Compute
$H_n = -\sum_v \tilde p_n(v) \log \tilde p_n(v)$ from the normalized
distribution $\tilde p_n(v) = c_n(v) / \sum_v c_n(v)$.
Allocate $\mathrm{cap}_n = C \cdot H_n / \sum_m H_m$ where $C$ is the
total enumeration budget (default 1\,M cycles).  Effect: arities with
flatter distributions (signal spread across many vertices) get more
budget than arities concentrated on hub vertices.

\paragraph{Test strategy.}
\begin{itemize}
  \item \textbf{Unit:} Hard-Concrete gate is in $[0,1]$ a.s.; gradient
    flows through $\log\alpha_k$ at temperature $\beta=2/3$;
    expected-L0 matches closed form within $10^{-6}$ on a synthetic
    weight vector (\texttt{test\_alpha\_k\_hard\_concrete.py}).
  \item \textbf{Unit:} ASHA pruner integration: mocked study,
    half-budget cohort pruned, surviving cohort completes
    (\texttt{test\_asha\_pruner.py}).
  \item \textbf{Unit:} Equal-info caps: known synthetic distributions
    (delta, uniform, geometric) produce expected ratios within 5\%
    (\texttt{test\_equal\_info\_caps.py}); Rust path tested in
    \texttt{cycle\_budget\_test.rs}.
  \item \textbf{Integration (smoke):} Bitcoin Alpha 1-seed, 80 epochs,
    \texttt{HSIKAN\_ARITY\_RANGE="2,3,4,5,6"} with
    \texttt{HSIKAN\_ALPHA\_K\_PRIOR=hard\_concrete}: must complete
    within performance budget below; learned gate vector must contain
    at least one $\bar g_k < 0.1$ (sparsity actually fires).
  \item \textbf{Performance:} median/IQR/worst over 5 iterations of
    enumeration wall time at $n=6$, $n=7$ on Bitcoin Alpha; RSS
    measured via \texttt{memray}.
  \item \textbf{Production-scale smoke (CLAUDE.md \S3):}
    1-seed Slashdot $c_2..c_6$ at the real $n_{\text{trials}}=1$ before
    any multi-seed paired run is queued.
\end{itemize}

\paragraph{Risk anticipation.}
\begin{itemize}
  \item \emph{Hard-Concrete temperature annealing.}  If $\beta$ cools
    too fast, $\alpha_k$ collapses to a single arity early and never
    explores; too slow, never sparsifies.  Mitigation: warmup of
    20\% of epochs at $\beta=2/3$, then linear to $\beta=1/3$.  Test
    on Bitcoin Alpha 1-seed before any multi-seed.
  \item \emph{ASHA on noisy plateau.}  Alpha Optuna trial 21 vs 23
    differed by $0.0001$.  ASHA at $r=4$ rungs may prune the eventual
    best.  Mitigation: minimum-rung budget set to $\geq$ 50\% of full
    training; promote-top-fraction set to $1/2$, not $1/3$.
  \item \emph{Equal-info caps interaction with existing
    \texttt{max\_k\_cap}.}  The existing per-arity \texttt{max\_cycles}
    cap (with aspirational ``push the cap into Rust'' comments) is a
    \emph{contract}.  New caps must be the \texttt{min} of (existing
    \texttt{max\_cycles}, equal-info $\mathrm{cap}_n$); never exceed.
    Test by constructing a fixture where equal-info would request more
    than \texttt{max\_cycles} and asserting the cap holds.
  \item \emph{Enumeration cost wall at $c_7$.}  Per memory
    \texttt{project\_rust\_cycle\_enum\_2026\_05\_02}, $c_4$ on
    Slashdot was 55\,M cycles in 4\,min.  $c_7$ is bounded but may
    require per-vertex top-$K$ + global-ABB scorer
    (\texttt{project\_abb\_global\_topk\_2026\_05\_10}) for tractability.
    Plan-budget assumes per-vertex top-$K$ active.  If the smoke at
    $c_6$ exceeds 12 min wall, halt and re-budget before $c_7$.
  \item \emph{Numerical stability of Hard-Concrete near $g_k=0$.}
    Avoid catastrophic cancellation in the gate-survival probability;
    use \texttt{log1p}/\texttt{expm1} forms.  Test against a
    \texttt{mpmath} reference at boundary $\log\alpha_k \in \{-20, 20\}$
    (CLAUDE.md \S5 numerical stability).
\end{itemize}

\paragraph{Performance-contract preservation.}
\textbf{Worst-case input.}  Slashdot at \texttt{HSIKAN\_ARITY\_RANGE
="2,3,4,5,6,7"} with default \texttt{max\_k\_cap}.  This is the
production-scale config the plan must survive; toy fixtures at $n=2$
are insufficient.  See CLAUDE.md \S3 production-scale smoke rule.

\paragraph{Empty-plan-dir hygiene.}
If this plan is abandoned before any of L1/L2/L3 is implemented,
delete \texttt{docs/plans/2026-05-13-cn-optimization-pattern/} so it
does not look like in-progress work.

\paragraph{Sequencing.}
Land in this order, one phase per session
(memory \texttt{feedback\_one\_phase\_per\_session}):
(1) L3 equal-info caps in Rust + Python dispatch + cache-fingerprint
extension; (2) L1 Hard-Concrete $\alpha_k$ with annealing schedule;
(3) L2 ASHA pruner wiring + first Bitcoin Alpha 30-trial sweep with
all three enabled.  Each phase ships with its tests, a smoke at
production scale, and a report under \texttt{reports/}.
